Overview

GIFT (Game Interface For Toys) is a program written in Python that allows sex toys to react to events that happen in-game in Skyrim or Fallout 4. It currently supports the following games:

  • The Elder Scrolls 5: Skyrim (LE, SE, AE, VR)

    • SexLab

    • Devious Devices

    • Toys & Love

    • SexLab Defeat

    • Naked Defeat

  • Fallout 4

    • Advanced Animation Framework (AAF)

    • AAF Violate

    • Devious Devices

    • Boston Devious Helper

Supported toys include:

  • Lovense

  • Buttplug.io (visit https://iostindex.com to check if your device is supported)

  • DG-Lab Coyote

  • Kizuna

In essence, the program runs in the background while you play, parsing the Papyrus log that Skyrim or Fallout 4 generates for in-game events such as getting hit, dying, experiencing DD vibrations or SexLab scenes, and reacts to them in almost real-time.

The script also supports Chaster, the keyholding and chastity platform. The current implementation requires a developer token (you can apply for this on Chaster), and it allows you to:

  • Add time to a lock.
  • Assign a task to the user of the lock (Task Extension)
  • Roll the dice (Dice Extension)
  • Spin the wheel of fortune (Wheel of fortune extension).

If using Chaster, the script spins the wheel when your character is knocked down via Defeat. When the whole party is defeated, adds a configurable amount of time to your lock. If using e-stim, applies a mild shock when hit, scaling in intensity as more damage is taken.

For a full overview of the latest supported features and devices, visit the main mod page: https://www.loverslab.com/files/file/22380-gift-game-interface-for-toys-skyrim-le-se-ae-vr-fallout-4/

Getting started

Choose your game:

  1. Install the following mods:

    • Skyrim Script Extender (SKSE)

    • PapyrusUtil SKSE Plugin

    • optional SexLab & SexLab Defeat

    • optional Devious Devices

    • optional Toys & Love

    • optional Naked Defeat

  2. Download and install Python (version 3.7 or later?) on your machine. Python is available for free here: https://www.python.org/downloads/windows/.

  3. Download GIFT from GitHub [direct download link] or LoversLab and extract the ZIP file to a folder. The unzipped contents should look like this: 

    .
├── common
│   ├── ...
├── GameInterfaceForToys.py
├── games
│   ├── fallout4
│   │   └── ...
│   └── skyrim
│       └── plugin
│           ├── scripts
│           │   ├── SkyrimToyInterfacePlayerScript.pex
│           │   └── source
│           │       └── SkyrimToyInterfacePlayerScript.psc
│           └── SkyrimToyInterface.esp
├── pattern_generator
│   ├── ...
├── settings.py
└── toys
    ├── ...

  4. Install the Skyrim mod SkyrimToyInterface.esp which is found in the games/skyrim/plugin folder. The easiest way to install the mod is to zip the contents of the folder (both the ESP file and the scripts folder!) into a new ZIP archive and then load & install it through a mod manager like Nexus Mod Manager or Vortex.

    TIP: You can verify that the mod is running by starting the game through your mod manager and typing 

    help SkyrimToyInterfaceControllerQuest

    in the in-game console. The mod should show up as a loaded quest like this: 

    QUST: SkyrimToyInterfaceController

  5. Enable debug tracing in your Skyrim.ini ? configuration file under the [Papyrus] section. Papyrus should have the flags bEnableLogging and bEnableTrace enabled. This allows Papyrus to output debug information into the log, which the script needs to function:

    [Papyrus]
bEnableLogging=1
bEnableTrace=1

  6. Open settings.py file in an ordinary text editor (like notepad++ or VS Code) and change the variables LOG_PATH and CHARACTER_NAME in the file:

    from common.constants import *
## Edit the following variables to configure this script

# Important variables to configure for all users:
LOG_PATH = '../Documents/My Games/Skyrim VR/Logs/Script/Papyrus.0.log' # Full path to the Papyrus log that the script should parse
CHARACTER_NAME = "Min"  # The name of your character.

    LOG_PATH Should point to the place on your hard disk where Skyrim outputs its Papyrus log file. The precise file path depends on your particular Skyrim installation, but for most installations the log can usually be found here: 

    LOG_PATH = 'C:/Users/YOUR_WINDOWS_ACCOUNT_NAME/Documents/My Games/Skyrim/Logs/Script/Papyrus.0.log'

    CHARACTER_NAME must correspond to the name of your in-game character, which is "Prisoner" by default. This ensures that the script can parse the correct information from the log file.

    Caution: settings.py is a PY (python script) file. Make sure your text editor does not save it as a TXT file like settings.py.txt!
