Overview

This python script allows sex toys to react to events that happen in-game in in Skyrim or Fallout 4. It currently supports the following games:

  • The Elder Scrolls 5: Skyrim (LE, SE, AE, VR)

    • SexLab

    • Devious Devices

    • Toys & Love

    • SexLab Defeat

    • Naked Defeat

  • Fallout 4

    • Advanced Animation Framework (AAF)

    • AAF Violate

    • Devious Devices

    • Boston Devious Helper

Supported toys include:

  • Lovense

  • Buttplug.io (visit https://iostindex.com to check if your device is supported)

  • Chaster

  • DG-Lab Coyote

  • Kizuna

In essence, the script runs in the background while you play, parses the Papyrus log that Skyrim or Fallout 4 generates for in-game events such as getting hit, dying, experiencing DD vibrations or SexLab scenes, and reacts to them in almost real-time.

The script also supports Chaster, the keyholding and chastity platform. The current implementation requires a developer token (You can apply for this on Chaster), and it allows you to:

  • Add time to a lock.
  • Assign a task to the user of the lock (Task Extension)
  • Roll the dice (Dice Extension)
  • Spin the wheel of fortune (Wheel of fortune extension).

If using Chaster, the script spins the wheel when your character is knocked down via Defeat. When the whole party is defeated, adds a configurable amount of time to your lock. If using e-stim, applies a mild shock when hit, scaling in intensity as more damage is taken.

For a full overview of the latest supported features and devices, visit the main mod page: https://www.loverslab.com/files/file/22380-gift-game-interface-for-toys-skyrim-le-se-ae-vr-fallout-4/

    LOG_PATH Should point to the place on your hard disk where Skyrim outputs its Papyrus log file. The precise file path depends on your particular Skyrim installation, but for most installations the log can usually be found here: 

    LOG_PATH = 'C:/Users/YOUR_WINDOWS_ACCOUNT_NAME/Documents/My Games/Skyrim/Logs/Script/Papyrus.0.log'

    CHARACTER_NAME must correspond to the name of your in-game character. This ensures that the script can parse the correct information from the log file.
